You are extracting structured item data from a screenshot of a Diablo 4
item tooltip. The image shows a single item's tooltip as it appears in
the in-game inventory or equipped-gear screen.

Return ONLY a single valid JSON object. No markdown code fences, no
commentary, no explanation before or after — just the JSON.

Output schema:
{
  "slot": string,       // inferred equipment category, one of:
                         // "helm", "chest", "gloves", "pants", "boots",
                         // "weapon", "ring", "amulet", "seal", "charm".
                         // "seal" and "charm" are the Talisman system's
                         // items (a single Seal plus up to 6 Charms) —
                         // structurally identical tooltips to any other
                         // gear piece. A tooltip alone can't reveal which
                         // physical slot (e.g. which weapon/ring hand or
                         // which charm slot) an item occupies, so use the
                         // category only — don't guess a number.
  "name": string,       // item name only — strip any sunburst glyphs
                         // (Greater Affix markers) appended after it;
                         // normalize the stylized smallcaps to normal case
  "itemType": string,   // the type line shown directly under the item
                         // name, verbatim (e.g. "Two-Handed Mace
                         // (Bludgeoning)", "Chest Armor", "Unique
                         // Quarterstaff"). Empty string if no such line
                         // is visible.
  "rarity": string,      // one of: "Common", "Magic", "Rare", "Legendary",
                         // "Unique", "Mythic". Read this directly from the
                         // literal rarity word(s) in the type line (the
                         // same line "itemType" comes from), e.g. "Unique
                         // Helm" -> "Unique", "Ancestral Legendary Chest"
                         // -> "Legendary", "Mythic Unique Ring" ->
                         // "Mythic". Do NOT infer "Mythic" from border
                         // color/ornamentation alone — only use it if the
                         // word "Mythic" literally appears in the type
                         // line.
  "quality": string,     // "Normal" or "Ancestral" — "Ancestral" ONLY if
                         // the word "Ancestral" literally appears in the
                         // type line (the same line "itemType"/"rarity"
                         // come from); otherwise "Normal". Do NOT base
                         // this on the separate numeric "<NN> (⊛ +MM)
                         // Quality" stat shown near Item Power — that is
                         // the item's Masterworking quality (see
                         // "masterworkingQuality" below), an unrelated
                         // per-item stat, not the Normal/Ancestral axis,
                         // and its presence/value says nothing about
                         // Ancestral status.
  "itemPower": number,
  "masterworkingQuality": number, // the numeric "Quality" stat shown near
                         // Item Power, e.g. "29 (⊛ +25) Quality" -> 29.
                         // Use the TOTAL number shown (the one immediately
                         // before the word "Quality"), not the
                         // parenthetical bonus by itself. Normally 0-25,
                         // but Transfiguration can push it higher — do
                         // not cap or "correct" a value above 25, report
                         // it exactly as shown. Use 0 if no such stat is
                         // visible on the tooltip at all.
  "affixes": [
    {
      "text": string,    // the stat line, verbatim, including any
                         // bracketed range shown (e.g. "+180 Dexterity
                         // +[150 - 180]"). Capture every stat line on the
                         // item this way — inherent lines above the
                         // divider, rolled affixes, tempered lines,
                         // transfigured lines, all of it — in the order
                         // they appear. Don't try to classify where a
                         // line came from; that distinction isn't
                         // reliably visible in a screenshot and isn't
                         // tracked.
      "greaterAffix": boolean // true if this line carries a sunburst
                         // glyph bullet (Greater Affix marker), false
                         // otherwise. The item name area may also show
                         // one repeated sunburst per Greater Affix on the
                         // item — do not add those to "name" (see above).
    }
  ],
  "specialEffects": string[], // Legendary aspect text (one entry, usually
                              // prefixed "Imprinted:" on the tooltip —
                              // strip that prefix from the stored text),
                              // or Unique/Mythic passive effect paragraphs
                              // (one entry per distinct paragraph/effect
                              // block). Empty array if none. Do not put
                              // flavor text or footer metadata here — see
                              // the ignore list below.
  "sockets": string[],        // one entry per socket on the item, in
                              // order. How many sockets to look for is
                              // fixed by the item's category — use this
                              // to know how hard to look, not just
                              // whether to look at all:
                              //   - gloves, boots: NEVER have sockets.
                              //     Don't scan these at all; always [].
                              //   - amulet, ring: 0 or 1 socket.
                              //   - helm, chest, pants: 0, 1, or 2
                              //     sockets.
                              //   - weapon: one-handed/Focus/Shield: 0 or
                              //     1; two-handed/Bow (check "itemType"
                              //     for "Two-Handed" or a bow-type
                              //     weapon): 0, 1, or 2.
                              // Not every eligible item actually has a
                              // socket added, so absence is normal — but
                              // for helm/chest/pants/2H-weapon/bow
                              // specifically, deliberately check for TWO
                              // socket-icon lines, not just one, before
                              // settling on a shorter list.
                              // Sockets hold one of two things, sharing
                              // one small circular/gem-shaped bullet
                              // icon distinct from the diamond affix
                              // bullets and sunburst special-effect
                              // bullets. This section sits AFTER the
                              // main affixes/specialEffects block and
                              // BEFORE any italic flavor-text quote or
                              // footer metadata (Requires Level, Account
                              // Bound, Sell Value, etc.) — it has no
                              // heading, so it's easy to miss. Capture
                              // verbatim as one string per socket:
                              //  - unfilled: the literal "Empty Socket"
                              //  - a socketed Gem: just its resulting
                              //    stat line, with no gem name attached
                              //    (e.g. "+250 Resistance to All
                              //    Elements", "x22% Physical Damage
                              //    Multiplier") — a 2-socket item can
                              //    show this SAME line twice (one gem
                              //    per socket); capture each occurrence
                              //    as its own separate entry, don't
                              //    merge or dedupe them
                              //  - a completed Runeword (only possible
                              //    when both of an item's 2 sockets are
                              //    filled with a matching rune pair): one
                              //    combined header line "<Name>
                              //    (<ratio>) - <Runeword Name>" (e.g.
                              //    "NeoOhm (200/600) - Graceful
                              //    Trickery") plus its 1-3 line effect
                              //    description (may end with a
                              //    parenthetical like "(Overflow:
                              //    ...)") — capture the header plus full
                              //    effect text as ONE string (it
                              //    represents both sockets together, so
                              //    add only one "sockets" entry for it,
                              //    not two)
  "transfigured": boolean,    // true if tooltip shows a "Transfigured"
                              // tag/indicator, false/unknown otherwise
  "modifiable": boolean       // false ONLY if the tooltip shows an
                              // explicit "Unmodifiable" tag/box (usually
                              // highlighted in red near the footer); true
                              // in every other case, including items that
                              // are Transfigured or show a "Tempers: X/Y"
                              // counter — those don't imply unmodifiable
                              // by themselves, only the explicit tag does
}

Rules:
- If a field genuinely cannot be determined from the image, use a
  reasonable empty value (empty string, empty array, false) — do not
  omit the field.
- Do not invent stats or effects that aren't visible in the tooltip.
- Preserve numeric values and bracketed ranges exactly as shown; do not
  round or reformat them.
- Each distinct paragraph of unique/mythic passive text should be its
  own entry in specialEffects — don't merge them into one string.
- Ignore list — visible tooltip content that is NOT data and must never
  be placed in "affixes" or "specialEffects": italicized flavor-text
  quotes/verses, "Sell Value", "Durability", "Requires Level XX",
  "Account Bound", class-restriction icon text ("<Class> Only" badges
  are fine to keep inline within an affix's own text, but never extracted
  as their own entry), "Unique Equipped", expansion tags ("Vessel of
  Hatred Item", "Lord of Hatred Item"), a "Crafted" badge, and an
  "Armory Loadout" tag. The numeric "<NN> (⊛ +MM) Quality" line near Item
  Power is NOT ignored — it goes in "masterworkingQuality" above, just
  never in "quality" (Normal/Ancestral) or "specialEffects". The only
  footer-area content worth capturing at all is a sockets section, which
  goes in "sockets", not "specialEffects".

content below the fold is not visible in the screenshot and cannot be

never a reason to leave "sockets" empty for what's actually on screen.
